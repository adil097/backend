Теория:
    Docker -  это инструмент для запуска приложений в контейнерах

    Контейнер - это виртуальная машина со ОС (линукс), 
    установленным прогами (библиотеки, и все такое),
    и сам код. НО они легче, быстрее и меньше самих ВМ

    Образ (Image) - это шаблон для создания контейнера. 
    В нем содержится все необходимое: операционная система, зависимости, библиотеки и само приложение. 
    Контейнеры запускаются на основе образов.
     

Установка: 
    установать wsl через powershell "wsl --install"
    Скачать докер с оф сайт
    Перезагрузка 



Как настроить докер для проекта? Нужно два (еще может .env) файла:
    Dockerfile - там конкретный Контейнер, указано что именно туда установать,
    какие команды запустить. 
    docker-compose.yml - какие контейнеры и сколько запустить

Команды для запуска докера и с работой с ним:
    docker compose up --build     # Собрать докер на основе docker-compose.yml
    docker compose up --build -d  # В фоне Собрать докер на основе docker-compose.yml



    docker build -t name .        # создать образ из Dockerfile
    docker images                 # список образов
    docker rmi <image_id>         # удалить образ

    docker run -p 8000:8000 name  # запустить контейнер из образа
    docker ps                     # запущенные контейнеры
    docker ps -a                  # все контейнеры
    docker stop <container_id>    # остановить
    docker start <container_id>   # запустить
    docker rm <container_id>      # удалить
